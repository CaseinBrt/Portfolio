# Customization Plan: Live Resume Template

## Project Overview

This is a **Next.js 16 + React 19 resume/portfolio template** built with Tailwind CSS 4, featuring:
- Fully responsive design with dark mode support
- Animated experience timeline
- Portfolio with project filtering
- Contact form
- Print-optimized PDF export
- Feature toggles for every section

---

## Your Next Steps (Immediate Actions)

### Step 1: Install Dependencies
```bash
bun install
```

### Step 2: Preview the Template
```bash
bun dev
```
Visit `http://localhost:3000` to see the default demo content.

### Step 3: Basic Customization (5-10 min)

Edit these 2 files to get a working personalized version:

1. **`src/data/profile.ts`** — Your personal information
   - Replace `name`, `title`, `email`, `location`
   - Update `summary` (2-4 sentences)
   - Add social links (github, linkedin, twitter)
   - Add a photo at `public/images/profile.jpg`

2. **`src/config/site.config.ts`** — Theme and features
   - Change `theme.primaryColor` to your preferred color (see comments for HSL values)
   - Toggle features on/off (disable sections you don't need)
   - Update SEO `meta` fields (siteUrl, keywords)

## Theme & Design Customization

### Primary Color (Quick Win)
In **`src/config/site.config.ts`** → `theme.primaryColor`:

| Color | HSL Value |
|-------|-----------|
| Blue (default) | `220 92% 50%` |
| Purple | `280 70% 50%` |
| Green | `150 70% 45%` |
| Orange | `30 90% 50%` |
| Pink | `330 70% 50%` |
| Teal | `180 70% 45%` |
| Red | `0 80% 50%` |

Generate your own: Find a color on [Tailwind CSS Colors](https://tailwindcss.com/docs/customizing-colors#using-css-variables) → convert to HSL.

### Enable/Disable Sections
In **`src/config/site.config.ts`** → `features`:

```typescript
features: {
  portfolio: true,        // Show portfolio section
  contactForm: true,      // Show contact form
  pdfDownload: true,      // PDF export button
  printButton: true,      // Print button
  skillBars: true,        // Visual skill bars
  animatedTimeline: true, // Experience animations
  certifications: true,   // Show certifications
  languages: true,        // Show languages section
  awards: true,          // Show awards section
  smoothScroll: true,    // Smooth scrolling
  sideNav: true,         // Desktop side navigation
}
```

Disable sections you don't need to simplify your resume.

### Navigation Order
Reorder sections by editing `sectionOrder` array in `site.config.ts`:

```typescript
sectionOrder: [
  'profile',
  'summary',
  'experience',
  'skills',
  'education',
  'certifications',
  'portfolio',
  'contact',
]
```

Rearrange or remove entries (but ensure corresponding sections exist in components).

### Social Media Platforms
Enable/disable platforms in `socialPlatforms` config. Set `enabled: false` for platforms you don't use.

## Advanced Customization (Optional)

### Add New Sections
Create new component in `src/components/resume/`:
1. Create `YourNewSection.tsx`
2. Follow the pattern: wrap content in `<Section>` and `<Container>`
3. Import your data from `src/data/`
4. Add to `src/app/page.tsx`
5. Add section ID to navigation if needed

### Custom Styling
- Global CSS: `src/app/globals.css`
- Component styles: Use Tailwind classes inline
- Theme CSS variables: `:root` selector in globals.css
- Dark mode: Already configured with `darkMode: 'class'`

### Contact Form Backend
The form posts to `/api/contact`. To receive emails:
- Use a service like Resend, Formspree, or EmailJS
- Modify API route in `src/app/api/contact/route.ts`
- Update `contactForm.endpoint` in config

### Images
- Profile photo: `public/images/profile.jpg` (recommended: 400x400px, face centered)
- Project thumbnails: `public/projects/` (recommended: 1200x630px for social sharing)
- Company/school logos: `public/logos/` (PNG with transparent background)
- Favicon: `public/favicon.ico`

### Analytics
Add Google Analytics, Plausible, or similar:
- Insert script in `src/app/layout.tsx` (inside `<body>`)
- Or use Next.js Script component with `strategy="afterInteractive"`

## Recommended Workflow

1. **Phase 1 — Content** (30 min)
   - Edit all 5 data files (`profile.ts`, `experience.ts`, `skills.ts`, `projects.ts`, `education.ts`)
   - Gather all your information before starting
   - Prepare images (profile photo, project screenshots)

2. **Phase 2 — Theme** (10 min)
   - Pick your primary color
   - Decide dark mode preferences
   - Choose font family (Inter, Roboto, Poppins, etc.)
   - Toggle features on/off

3. **Phase 3 — Preview & Iterate** (15 min)
   - Run `bun dev`
   - Check all sections on mobile + desktop
   - Test print view at `/print`
   - Adjust content/spacing as needed

4. **Phase 4 — Polish** (15 min)
   - Refine summary and highlights
   - Ensure achievements have metrics
   - Update project descriptions to be compelling
   - Add finishing touches (honors, courses, etc.)

5. **Phase 5 — Deploy** (10 min)
   - Run `bun build` to validate production build
   - Fix any type errors (`bun typecheck`)
   - Fix any lint issues (`bun lint`)
   - Deploy to Vercel, Netlify, or any static host

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Images not showing | Check paths: must be in `public/` folder, reference as `/images/xxx.jpg` |
| Color not changing | Ensure you changed `theme.primaryColor` in site.config.ts and saved |
| Section still shows after disabling | Some sections (e.g., certifications) may also need toggles in component |
| Build errors | Run `bun typecheck` to identify TypeScript issues |
| Layout broken | Check Tailwind classes, ensure Tailwind is configured correctly |

## Pro Tips

1. **Metrics matter** — Add numbers to achievements (e.g., "Increased conversions by 25%")
2. **Be concise** — Keep summaries under 4 sentences; use bullet points
3. **Quality over quantity** — 3-4 strong projects > 10 weak ones
4. **Active voice** — Start achievements with action verbs (Led, Built, Reduced, Optimized)
5. **Proofread** — Check spelling/grammar; have someone review your content
6. **Mobile-first** — Preview on phone; ensure readibility on small screens
7. **Keep it current** — Remove outdated tech or irrelevant experience
